Neural Cellular Automata

Genotype-Conditioned
Self-Organising Textures

A neural cellular automaton whose growth behaviour is governed by a discrete gene vector, enabling multiple co-existing species to emerge from a single shared update rule.

Anonymous Author
Submitted 2025 · Interactive article
Abstract. We present Gene NCA, a neural cellular automaton (NCA) augmented with an explicit genotype: a small binary vector appended to each cell's state that remains frozen during the learned update step. The genotype breaks the symmetry of a homogeneous field, allowing cells of different lineages to grow into visually distinct textures despite executing identical local rules. We describe the architecture, the perception mechanism, and the stochastic alive-masking scheme that together enable stable, multi-species co-existence. An interactive WebGL demo embedded in this article allows the reader to paint genotypes onto the canvas and observe emergent growth in real time.

1 · Motivation

Biological development is a remarkable example of a system where identical local rules—encoded in the genome shared by every cell—give rise to globally coherent, spatially heterogeneous structures. A liver cell and a neuron carry the same DNA; it is the expression state of that DNA, shaped by developmental history and local signalling, that determines cell identity.

Neural Cellular Automata (NCA) 1 offer a differentiable model of this phenomenon: each cell in a grid runs the same learned update rule, communicating only with immediate neighbours through a fixed perception kernel. Trained end-to-end, such systems can grow, maintain, and regenerate target patterns starting from a single seed cell.

A natural extension is to ask: can a single trained network support multiple distinct species simultaneously, distinguishable only by a small internal flag? We explore this idea by appending a fixed binary gene vector g ∈ {0,1}4 to each cell. The gene is readable by the update network as part of the cell's perception, but is never modified by the update step—it functions as an immutable lineage marker.

2 · Model Architecture

Cell State

Each cell carries a state vector x ∈ ℝ24. The first three channels are interpreted as RGB colour; channel 3 is an alpha (opacity) value that determines cell liveness; channels 4–19 are latent hidden channels; and channels 20–23 form the immutable gene vector g.

Perception

Before the learned update rule is applied, each cell aggregates neighbourhood information through three fixed, hand-crafted convolutional filters applied independently to each of the 24 state channels. This yields a perception vector of dimension 24 × 4 = 96 (the original state concatenated with three filtered views).

Update Rule

The perception vector feeds a small two-layer network—the "update rule"—parameterised by two 1×1 convolutions. Following the original NCA paper, the final layer is initialised to zero so that the network begins in a do-nothing state.

// Perception
p = concat( x, sobel_x(x), sobel_y(x), laplacian(x) ) // ℝ⁹⁶

// Update rule
h = ReLU( W₁ · p + b₁ ) // ℝ²⁵⁶
Δ = W₂ · h // ℝ²⁰ (no bias, zero-init)

// Masked residual update
x[:20] = x[:20] + Δ · mask_stoch · mask_alive
x[20:] = g // gene preserved unchanged

3 · Stochastic Cell Updates and Alive Masking

Two binary masks gate the update at each step.

Stochastic update mask. With probability 0.5, each cell skips its update entirely this step, independently of its neighbours. This asynchrony prevents the automaton from exploiting global phase information and improves robustness to perturbation—a biological parallel to the noisy, asynchronous nature of gene expression.

Alive mask. A cell is considered alive if the maximum alpha value in its Moore neighbourhood (a 3×3 patch) exceeds 0.1. Cells that fail this test have their delta zeroed out—they neither receive updates nor propagate state to neighbours. This creates a sharp growth frontier: interior, well-supported cells grow actively; isolated stragglers die.

mask_stoch = Bernoulli(0.5) // per cell, per step
mask_alive = max_pool(α, 3×3) > 0.1 // α = channel 3

4 · The Gene Vector as Lineage Marker

The key innovation of Gene NCA over the original growing NCA is the introduction of the gene suffix. Because every cell executing the update rule reads its own gene channels as part of the 96-dimensional perception vector, the learned weights implicitly partition their behaviour across the 24 = 16 possible gene values (minus all-zeros, giving 15 distinct species).

Crucially, the network is trained on all species simultaneously. The gene does not select a separate sub-network; it modulates a single shared computation. This is analogous to a eukaryotic cell: identical machinery, different regulatory inputs.

Because the gene is never overwritten by the update step, it propagates perfectly through cell division (painting). A patch seeded with gene 1001 will always produce 1001 cells, regardless of what happens in neighbouring regions.

5 · Interactive Demo

The figure below runs Gene NCA live in your browser using WebGL2. The neural network forward pass—including the depthwise perception convolutions and the two matrix multiplications—is implemented as GLSL fragment shaders executing on your GPU. No server round-trips occur after the model weights are loaded.

Load your trained art_4.pth weights file, or press Run demo to initialise with random weights (which will not produce trained behaviour, but demonstrates the simulation pipeline). Then select a gene from the palette and paint onto the canvas.

6 · Implementation Notes

State texture packing

WebGL2 RGBA32F textures store four float channels per texel. To represent the 24-channel cell state we allocate 6 texels per cell. A grid of 128×128 cells is therefore stored in a 768×128 floating-point texture. Two such textures are maintained in a ping-pong buffer arrangement: one is read during the NCA step shader, the other is written, and the roles swap each frame.

Weight layout

The W₁ matrix (256×96) and W₂ matrix (20×256) are stored as RGBA32F textures, with groups of four input features packed per texel. The GLSL shader loops over the hidden dimension and accumulates dot products in registers, reading weight texels with texture() calls. On modern GPU hardware this inner loop is partially unrolled by the driver and executes in a handful of microseconds per frame.

Weight loading

PyTorch .pth files are ZIP archives containing a pickle stream (data.pkl) and raw tensor buffers (data/N). We implement a minimal ZIP and pickle parser in plain JavaScript, extracting the three relevant tensors by shape—no server-side conversion is needed. Tensors are identified as: W₁ weight at shape [256, 96], W₁ bias at shape [256], and W₂ weight at shape [20, 256].

7 · Results and Observations

When trained jointly across all 15 gene values, the network learns to produce markedly different texture statistics per gene while sharing all parameters. Gene combinations that differ in only one bit often produce visually related but distinguishable patterns, suggesting the network has learned a smooth interpolation in gene space rather than 15 entirely independent solutions.

At the boundary between two gene regions—painted manually in the interactive demo— neither species colonises the other. The alive mask prevents dead cells from spontaneously activating, and the gene vector prevents existing cells from changing species. This gives boundaries a sharp, stable quality reminiscent of tissue compartment boundaries in developmental biology.
